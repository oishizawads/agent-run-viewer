# Agent Run Viewer

A small public web app that visualizes **AI agent execution logs** — breaking a run
down into **steps, tool calls, successes, and failures**. Built to demonstrate log
design and basic observability for agent systems.

## 目的 / Purpose

Show how an agent run can be decomposed into a timeline of steps, with tool
invocations, durations, and errors surfaced separately. The viewer reads a JSON
log and renders:

- **Run List** — all runs with status, duration, step count, and tool-call count
- **Summary** — per-run aggregates (total duration, steps, tool calls, failures)
- **Timeline** — steps in chronological order, with failed steps emphasized
- **Tool Calls** — per-tool call counts/failures/time + every tool invocation
- **Errors** — failed steps with their error messages

## 主要機能 / Features

- Loads a JSON log at runtime (`/data/runs.json`) — swap the file to view other logs
- Chronological timeline with type and status badges
- Failed steps highlighted in red across Timeline, Tool Calls, and Errors
- Aggregates: total duration (wall-clock, with step-sum fallback), tool-call
  counts, failed/skipped counts, per-tool stats
- Tolerant parsing: empty logs, broken JSON, malformed entries, and very long
  logs all render an empty/error state instead of crashing
- Responsive, no horizontal scroll on mobile width
- Static export (`output: "export"`) — deployable to any static host

## 使用技術 / Tech Stack

- [Next.js](https://nextjs.org/) 16 (App Router, static export)
- TypeScript
- Tailwind CSS v3
- Vitest + @testing-library/react (tests)

## データの出所 / Data Source

All logs are **synthetic**. They are generated by
`scripts/generate-sample-data.mjs` into `data/runs.json` and synced to
`public/data/runs.json` before dev/build. **No real logs, secrets, or PII** are
used. To regenerate: `npm run gen-data`.

The schema (see `src/lib/types.ts`):

```ts
Run  { id, name, agent, model, startedAt, endedAt, status, tags?, steps[] }
Step { id, name, type, status, startedAt, endedAt, durationMs,
       tool?, toolInput?, toolOutput?, error?, message? }
```

`status`: `completed | failed | partial` (run) / `success | failed | skipped` (step)
`type`:  `plan | reasoning | tool | output | error`

## ローカル実行手順 / Local Setup

Requires Node 18.18+ (developed on Node 24).

```bash
npm install        # install dependencies
npm run dev        # start dev server (syncs data first via predev hook)
# open http://localhost:3000
```

Other scripts:

```bash
npm run build      # static export to ./out (prebuild syncs data)
npm run start      # serve the production build
npm run lint       # eslint
npm run typecheck  # tsc --noEmit
npm test           # vitest run
npm run gen-data   # regenerate sample data + sync
```

To view your own logs: replace `data/runs.json` (or add more `.json` files in
`data/`) and re-run `npm run dev` / `npm run build`. The `predev`/`prebuild`
hooks copy `data/*.json` into `public/data/` so the app can fetch them at runtime.

## スクショ置き場 / Screenshots

Screenshots live in [`assets/`](./assets) (e.g. `run-list.png`, `timeline.png`,
`errors.png`). Add yours there.

## デプロイ先 / Deployment

Configured for **static export** (`output: "export"` in `next.config.mjs`),
outputting to `./out`. Deploy that directory to any static host:

- **Vercel**: import the repo; the framework preset and `next build` produce a
  static export automatically.
- **GitHub Pages**: publish the `./out` folder (e.g. via an Actions workflow).
  `trailingSlash: true` is set for compatibility.

No environment variables or API keys are required (see `.env.example`).

## 制限事項 / Limitations

- MVP only: no auth, no database, no backend API, no real-time streaming.
- Logs are read from a static JSON file at runtime; changing logs on a deployed
  static site requires redeploying (or swapping the file on the host).
- Run selection is in-memory (no deep-linkable URL per run).
- Time values are rendered in UTC for deterministic display.
